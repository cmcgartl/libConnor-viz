A High-Performance Memory Allocator

Custom malloc / free / realloc implementation in C, using segregated free lists, immediate coalescing, and bidirectional allocation, with optional thread safety and benchmarked against libc.

98% of libc throughput
93% of libc utilization
Heap Visualizer Benchmark Dashboard

Segregated Free Lists

13 size-class buckets with best-fit search for optimal memory utilization. Large allocations are split from the back to reduce fragmentation.

Immediate Coalescing

Adjacent free blocks are merged on every free() call using boundary tags (header/footer), preventing external fragmentation.

Thread Safety

Optional recursive mutex locking around all allocator entry points. Verified with an 8-thread stress test doing 80K concurrent operations.

Benchmarked Against libc

Tested across 13 real-world memory traces. Achieves 93% average utilization and up to 98% of libc throughput on allocation-heavy workloads.

Heap Visualizer

Step through 10,000 heap events and watch blocks get allocated, freed, and coalesced in real time. See fragmentation patterns as they develop.

64-bit Native

16-byte aligned allocations on 64-bit architectures. Minimum 32-byte blocks with 8-byte headers and footers for boundary tag metadata.
